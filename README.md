# Team LEGO — موقع الفريق

## الملفات
- `index.html` — الصفحة الرئيسية (كل الأقسام)
- `style.css` — كل التنسيقات
- `script.js` — كل التفاعل (نضيف بالكامل، من غير أي كود مشفّر)
- `logo.png` — شعار الفريق
- `legal.html` — سياسة الخصوصية وشروط الاستخدام
- `404.html` — صفحة الخطأ المخصصة (تظهر تلقائيًا حسب إعدادات الاستضافة)
- `robots.txt` + `sitemap.xml` — ملفات أساسية لظهور الموقع في جوجل
- `blog.html` + فولدر `blog/` — المدونة التقنية (3 مقالات جاهزة)
- `server-example.js` — سيرفر جاهز لربط قسم "المساعد الذكي" بـ Claude فعليًا

## ⚠️ قبل ما ترفع الموقع أونلاين
دوّر على النص `your-domain.example.com` في الملفات دي وغيّره لدومينك الحقيقي:
`index.html`, `legal.html`, `blog.html`, ملفات `blog/*.html`, `robots.txt`, `sitemap.xml`.

## تفعيل صفحة 404 حسب الاستضافة
- **Netlify:** الملف `404.html` في الجذر بيشتغل تلقائيًا.
- **GitHub Pages:** نفس الشيء، لازم يكون اسمه `404.html` بالظبط في الجذر (زي ما هو).
- **استضافة مشتركة (cPanel):** غالبًا محتاج تحدد المسار في إعدادات الـ Error Pages من لوحة التحكم.

## تشغيل الموقع محليًا
افتح `index.html` بـ Live Server في VS Code (زرار يمين على الملف → Open with
Live Server).

## تشغيل "المساعد الذكي" فعليًا (مش الديمو)
بالديفولت، قسم الشات بيرد بردود تجريبية (Demo) عشان الموقع يشتغل فورًا من
غير سيرفر. عشان يبقى مربوط بـ Claude فعليًا:

1. افتح `server-example.js` واتبع التعليقات جواه خطوة بخطوة.
2. هتحتاج مفتاح API من https://console.anthropic.com/settings/keys
3. ترفع السيرفر على استضافة بتدعم Node.js (Render, Railway, Vercel...).
4. في `script.js`، دوّر على `AI_BACKEND_URL` و `USE_DEMO_MODE` وغيّرهم
   لعنوان السيرفر بتاعك و `false`.

المفتاح السري لازم يفضل بس على السيرفر، وميتحطش أبدًا جوه `script.js` أو
أي ملف بيشتغل في متصفح الزائر.
